Abstract

Motivação:

O SOAP foi criado com o intuito de ser um protocolo de comunicação simples (pouco exigente), baseado em XML e destinado à troca de informação tipada e estruturada entre aplicações distribuídas e descentralizadas. De forma a não comprometer a independência em relação às plataformas de desenvolvimento, o SOAP não define um protocolo de transporte para o envio de mensagens, apesar de possuir uma descrição de como deve ser utilizado sobre HTTP. A existência desta descrição deve-se ao facto do HTTP ser o protocolo standard para a Internet. No entanto, o SOAP não se encontra vinculado ao HTTP, podendo utilizar uma grande variedade de outros protocolos. Por exemplo: FTP - File Transfer Protocol, SMTP - Simple Mail Transfer Protocol, BEEP - Blocks Extensible Exchange Protocol. implementação de Web Services em PHP utilizando a classe NuSoap. Nos ultimos tempos a criação de web services tem sido cada vez mais comum. Para construir tal tem sido usado mais frequentemente WebServices com NuSOAP. Tal é um conjunto de classes PHP que permite aos programadores criar e consumir serviços Web baseados em SOAP 1.1, 1.1 e WSDL HTTP 1.0/1.1.

Objectivo :

Neste trabalho de casa o objectivo era criar um web service sobre um ficheiro de inquirições que oferecesse serviços tais como :

  • Contar quantas vezes um certo nome se encontra no ficheiro xml
  • Listar todos os ids que aparecem junto ao nome enviado
    • Para a construção deste serviço, foi criado um servidor e um cliente. O servidor acede ao ficheiro xml e envia os dados para o cliente.

    Notas importantes :

  • Primeiro, quando a instância do soapclient é criado, o parâmetro especificado é o URL para o serviço. No meu caso, server.php é acedido a partir http://localhost:8888/TPC4/server.php
  • Em segundo lugar, ao chamar o serviço, o primeiro parâmetro é o nome do serviço. Este deve corresponder ao nome case-sensítive de um método disponível no servidor. Neste caso, ele deve corresponder aos métodos registrados no server.php.
